Belajar HTML Dasar

Memahami struktur, semantik, dan elemen HTML — bukan sekadar hapal, tapi paham kenapa.

1. Elemen Teks

Tag Teks yang Sering Dipakai

Ini adalah paragraf biasa dengan tag <p>. Browser otomatis memberi jarak atas-bawah antar paragraf.

strong = teks PENTING secara semantik (screen reader menekankan ini). Berbeda dengan b yang hanya visual bold tanpa makna.

em = teks yang ditekankan dalam kalimat (screen reader mengubah intonasi). Berbeda dengan i yang hanya visual italic (untuk judul buku, istilah teknis, dll).

Kode inline: console.log('hello') — pakai <code> untuk kode dalam teks.

Subscript: H2O  |  Superscript: x2 + y2

Highlighted text pakai <mark> — berguna untuk menandai hasil pencarian atau teks penting.

Harga lama: Rp 200.000 → Harga baru: Rp 150.000 (<s> untuk teks yang sudah tidak relevan)

HTML Entities — Karakter Khusus

Karakter yang tidak bisa langsung ditulis di HTML:

  • &lt; → < (less than)
  • &gt; → > (greater than)
  • &amp; → & (ampersand)
  • &copy; → © (copyright)
  • &nbsp; → non-breaking space
  • &mdash; → — (em dash)
  • &hellip; → … (ellipsis)

2. List

Unordered List <ul>

Untuk daftar yang urutannya tidak penting:

  • HTML — struktur
  • CSS — tampilan
  • JavaScript — interaksi

Pakai <ul> untuk fitur, bahan, navigasi, dll.

Ordered List <ol>

Untuk daftar yang urutannya penting:

  1. Tulis HTML dulu
  2. Tambahkan CSS
  3. Baru JavaScript

Pakai <ol> untuk langkah, ranking, prosedur.

Description List <dl>

Untuk pasangan istilah–definisi:

HTML
HyperText Markup Language — struktur halaman web
CSS
Cascading Style Sheets — tampilan halaman web
JS
JavaScript — interaktivitas halaman web

Nested List

3. Tabel

Kapan Pakai Tabel?

Tabel dipakai untuk data tabular — data yang memang berbentuk baris dan kolom. Jangan pakai tabel untuk layout halaman (dulu sering dilakukan, sekarang salah secara semantik — gunakan Flexbox atau Grid).

Perbandingan Elemen HTML Semantik vs Non-Semantik
Elemen Tipe Makna Contoh Penggunaan
<header> Semantik ✅ Kepala halaman/artikel Logo, navigasi utama
<nav> Semantik ✅ Navigasi utama Menu, breadcrumb
<main> Semantik ✅ Konten utama halaman Konten unik per halaman
<article> Semantik ✅ Konten mandiri Postingan blog, berita, produk
<section> Semantik ✅ Kelompok konten bertema Bagian dengan heading sendiri
<aside> Semantik ✅ Konten sampingan Sidebar, tip, iklan terkait
<footer> Semantik ✅ Kaki halaman/artikel Copyright, link tambahan
<div> Non-semantik ❌ Wadah generik (block) Hanya untuk styling/layout
<span> Non-semantik ❌ Wadah generik (inline) Styling sebagian teks
Selalu pilih elemen semantik jika tersedia. Pakai <div> dan <span> hanya jika tidak ada elemen semantik yang sesuai.

5. Gambar & Link

Gambar <img>

Selalu isi alt! Walau hanya alt="" untuk gambar dekoratif.

Link <a>

  • MDN Web Docs ↗ — target="_blank" + rel="noopener noreferrer" untuk keamanan
  • Kembali ke bagian Teks ↑ — href="#id" untuk link ke bagian di halaman yang sama
  • Kirim Email — href="mailto:..." membuka aplikasi email
  • Hubungi via Telepon — href="tel:..." di mobile langsung buka dialer
  • Download File — atribut download memaksa browser download file

📌 Catatan Penting

  • Hanya 1 <h1> per halaman
  • Jangan skip level heading (h1 → h3)
  • Selalu isi atribut alt pada gambar
  • Pakai elemen semantik, bukan sekadar <div>
  • <label for> harus terhubung ke id input
  • GET untuk read, POST untuk write/sensitif
