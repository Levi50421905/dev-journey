1. Cascade — Aturan Mana yang Menang?

"Cascading" dalam CSS berarti aturan diterapkan bertingkat. Ketika ada konflik (dua aturan menyasar elemen yang sama), CSS punya sistem untuk menentukan mana yang dipakai.

Urutan Prioritas (dari terendah ke tertinggi):

  1. Browser default styles — style bawaan browser (h1 besar, link biru, dll)
  2. External stylesheet — file CSS yang di-link (<link rel="stylesheet">)
  3. Internal stylesheet — CSS di dalam tag <style>
  4. Inline style — style="..." langsung di elemen HTML
  5. !important — paksa menang (hindari — membuat CSS susah di-debug)

Demo Cascade

Paragraf ini warnanya ditentukan oleh aturan yang paling spesifik. Lihat CSS-nya untuk memahami mana yang menang. 

/* Aturan 1 — tag selector */
p { color: gray; }

/* Aturan 2 — class selector (lebih spesifik) */
.cascade-demo { color: blue; }

/* Aturan 3 — di bawah = menang jika sama spesifik */
/* .cascade-demo { color: green; } */

2. Specificity — Seberapa Spesifik Selector?

Specificity adalah "skor" yang menentukan selector mana yang menang saat ada konflik. Skor dihitung dari tipe selector yang dipakai.

Selector Contoh Skor
Inline style style="color: red" 1,0,0,0
ID selector #judul 0,1,0,0
Class / Pseudo-class / Attribute .aktif :hover [type] 0,0,1,0
Tag / Pseudo-element p h1 ::before 0,0,0,1
Universal / Combinator * + ~ > 0,0,0,0

Contoh Perhitungan:

p 0,0,0,1
.card p 0,0,1,1
#main .card p 0,1,1,1
#main .card p strong 0,1,1,2 Menang!
💡 Tips: Hindari ID selector dan !important untuk styling — susah di-override. Pakai class saja, buat selector se-sederhana mungkin.

4. Positioning — Menentukan Posisi Elemen

static

Default. Mengikuti aliran normal dokumen. top/left/right/bottom tidak bekerja.

relative

Seperti static, tapi bisa digeser dari posisi normalnya. Ruang asli tetap ada. Sering dipakai sebagai kontainer untuk absolute.

Normal
Digeser 10px
Normal
absolute

Keluar dari aliran dokumen. Posisi relatif ke parent terdekat yang position != static. Jika tidak ada → relatif ke <html>.

absolute

Parent (position: relative)

fixed

Menempel di viewport. Tidak bergerak saat scroll. Cocok untuk: navbar sticky, tombol kembali ke atas, notifikasi.

Navbar di halaman ini pakai position: sticky.

sticky

Campuran relative + fixed. Normal mengikuti aliran, lalu "menempel" saat mencapai threshold yang ditentukan. Cocok untuk: header tabel, sidebar.

💡 z-index: Menentukan tumpukan elemen. Nilai lebih tinggi = tampil di depan. Hanya bekerja pada elemen yang position != static.
Konvensi umum: navbar 100, modal 200, tooltip 300.

5. Pseudo-class & Pseudo-element

Pseudo-class :

Kondisi atau state elemen. Satu titik dua.

  • :hover — mouse di atas elemen
  • :focus — elemen sedang aktif
  • :active — saat diklik
  • :visited — link sudah dikunjungi
  • :first-child — anak pertama
  • :last-child — anak terakhir
  • :nth-child(n) — anak ke-n
  • :nth-child(even) — anak genap
  • :nth-child(odd) — anak ganjil
  • :not(.class) — bukan elemen ini
  • :disabled — input yang di-disable
  • :checked — checkbox/radio yang diceklis

Pseudo-element ::

Bagian dari elemen. Dua titik dua.

  • ::before — konten sebelum elemen
  • ::after — konten setelah elemen
  • ::first-line — baris pertama teks
  • ::first-letter — huruf pertama
  • ::placeholder — teks placeholder input
  • ::selection — teks yang diseleksi user

6. Transisi & Animasi Dasar

transition membuat perubahan style berjalan mulus daripada langsung berubah secara instan. 

/* Format: property | durasi | timing-function | delay */
transition: background 0.3s ease;
transition: all 0.2s ease-in-out;

/* Timing functions: */
ease        → lambat-cepat-lambat (default)
linear      → konstan
ease-in     → mulai lambat
ease-out    → akhir lambat
ease-in-out → lambat di awal dan akhir

CSS Animation dengan @keyframes:

@keyframes nama-animasi {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

.elemen {
  animation: nama-animasi 0.5s ease forwards;
  /* nama | durasi | timing | fill-mode */
}
