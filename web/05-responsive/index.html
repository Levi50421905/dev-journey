1. Media Query — @media

Media query memungkinkan kita menerapkan CSS hanya pada kondisi tertentu — biasanya berdasarkan lebar layar.

Sintaks Dasar

/* Terapkan CSS ini HANYA jika lebar layar minimal 768px */
@media (min-width: 768px) {
  .container {
    max-width: 1100px;
    margin: 0 auto;
  }
}

/* Terapkan CSS ini HANYA jika lebar layar maksimal 767px */
@media (max-width: 767px) {
  .nav-menu {
    display: none; /* sembunyikan menu di mobile */
  }
}

/* Gabungan — antara 768px dan 1023px */
@media (min-width: 768px) and (max-width: 1023px) {
  /* hanya untuk tablet */
}

Mobile-First vs Desktop-First

Mobile First ✅ 
/* CSS default: untuk mobile */
.grid { display: block; }

/* Tambah untuk tablet ke atas */
@media (min-width: 768px) {
  .grid { display: grid; grid-template-columns: 1fr 1fr; }
}

/* Tambah untuk desktop */
@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

Direkomendasikan — mulai dari simple, tambah kompleksitas

Desktop First ❌ 
/* CSS default: untuk desktop */
.grid { display: grid; grid-template-columns: repeat(3, 1fr); }

/* Kurangi untuk tablet */
@media (max-width: 1023px) {
  .grid { grid-template-columns: 1fr 1fr; }
}

/* Kurangi lagi untuk mobile */
@media (max-width: 767px) {
  .grid { display: block; }
}

Tidak direkomendasikan — lebih berat, lebih sulit di-maintain

Resize browser untuk melihat perubahan:

📲 Mobile (< 768px) 📱 Tablet (768–1023px) 🖥️ Desktop (≥ 1024px)

2. Breakpoint — Titik Perubahan Layout

Breakpoint adalah nilai lebar layar di mana layout berubah. Tidak ada aturan baku — tapi ini yang paling umum dipakai:

320px
Mobile kecil iPhone SE, Android lama
480px
Mobile besar iPhone Plus, Android besar
768px
Tablet portrait iPad, tablet Android
1024px
Tablet landscape / Laptop kecil iPad landscape, laptop 13"
1280px
Desktop Monitor standar, laptop 15"
1536px+
Desktop besar / 4K Monitor 27"+, layar 4K
💡 Tips: Jangan terpaku pada breakpoint tertentu. Lebih baik tambahkan breakpoint saat layout terlihat rusak — bukan berdasarkan ukuran device spesifik. Resize browser sambil lihat hasilnya.

3. Fluid Typography & Spacing

Daripada mendefinisikan ukuran font berbeda di setiap breakpoint, kita bisa pakai fungsi clamp() untuk ukuran yang otomatis menyesuaikan secara mulus.

clamp(min, ideal, max)

/* font-size: minimal 1rem, ideal 4% lebar viewport, maksimal 2.5rem */
h1 { font-size: clamp(1.5rem, 4vw, 2.5rem); }

/* Tanpa clamp — harus tulis media query */
h1 { font-size: 2.5rem; }
@media (max-width: 768px) { h1 { font-size: 1.8rem; } }
@media (max-width: 480px) { h1 { font-size: 1.5rem; } }

/* Satuan viewport: */
/* vw = % dari lebar viewport */
/* vh = % dari tinggi viewport */
/* 1vw = 1% lebar layar */

Judul Fluid (clamp)

Sub Judul Fluid

Teks biasa yang ukurannya juga menyesuaikan secara mulus dengan lebar layar.

Satuan Relatif untuk Spacing

/* Pakai rem untuk spacing yang konsisten */
.section  { padding: 3rem 0; }       /* 48px di base 16px */
.card     { padding: 1.5rem; }       /* 24px */
.gap      { gap: 1rem; }             /* 16px */

/* Pakai % untuk lebar yang fleksibel */
.container { max-width: 1100px; width: 90%; margin: 0 auto; }
/* width: 90% → selalu ada 5% ruang di kiri-kanan */

4. Gambar Responsif

CSS Dasar

/* Gambar tidak melebihi lebar container */
img {
  max-width: 100%;
  height: auto;    /* pertahankan aspect ratio */
  display: block;  /* hilangkan gap bawah */
}

/* Gambar mengisi container dengan aspect ratio terjaga */
.img-cover {
  width: 100%;
  height: 200px;
  object-fit: cover;   /* crop ke tengah */
  /* object-fit: contain → fit tanpa crop */
  /* object-fit: fill   → stretch (distorsi) */
}
cover

object-fit: cover — crop ke tengah

contain

object-fit: contain — fit tanpa crop

fill

object-fit: fill — stretch (hindari)

Gambar Berbeda per Ukuran Layar — <picture>

<picture>
  <source media="(min-width: 1024px)" srcset="gambar-desktop.jpg">
  <source media="(min-width: 768px)"  srcset="gambar-tablet.jpg">
  <img src="gambar-mobile.jpg" alt="Deskripsi gambar">
</picture>

<!-- Browser pilih source yang sesuai, fallback ke <img> -->
